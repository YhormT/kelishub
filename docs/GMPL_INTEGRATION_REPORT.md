## 1. Executive summary

Kellishub has implemented integration with the GMPL agent order API as documented (`POST /api/orders/agents/new`). The request format (authentication, multipart fields, and network provider) matches your specification.

**All GMPL submissions are currently failing** because the hostname **`api.gmpl.com` does not resolve in public DNS**. This affects Kellishub production (Render), local development, and manual testing. The failure occurs before any HTTP request reaches GMPL's servers.

**Action required from GMPL:** Provide the correct, publicly reachable **production API base URL**, confirm the service is live, and validate our Excel file format and any status/webhook endpoints.

## 2. Documented GMPL API (as provided)

| Item | Specification |
|------|----------------|
| **Endpoint** | `POST /api/orders/agents/new` |
| **Base URL (documented)** | `https://api.gmpl.com` |
| **Authentication** | `Authorization: Bearer <api-key-secret>` |
| **Alternate auth** | `x-clerk-api-key: <api-key-secret>` |
| **Content-Type** | `multipart/form-data` |
| **Form fields** | `orderFile` (Excel file), `network_provider` (e.g. `mtn`) |

```bash
curl -X POST \
  -H "Authorization: Bearer <your-secret>" \
  -F "orderFile=@order.xlsx" \
  -F "network_provider=mtn" \
  https://api.gmpl.com/api/orders/agents/new
```

## 3. What Kellishub implemented

### 3.1 HTTP client

Kellishub sends requests as follows:

- **URL:** `{GMPL_API_URL}/api/orders/agents/new`
  - Default `GMPL_API_URL`: `https://api.gmpl.com` (base only; path is appended in code)
- **Headers:** Both `Authorization: Bearer <key>` and `x-clerk-api-key: <key>`
- **Body:** `multipart/form-data` with:
  - `orderFile` — `.xlsx` file
  - `network_provider` — lowercase (e.g. `mtn` for MTN orders)

### 3.2 Order flows using GMPL

| Flow | Description |
|------|-------------|
| **Auto-export** | Pending MTN wallet orders are batched (one batch per purchaser order), Excel is generated automatically from order data, and submitted to GMPL on a schedule and when new orders arrive. |
| **Manual export** | Admin can export pending MTN orders from the dashboard; Excel is built from pending items and submitted to GMPL. |
| **Batch retry** | Failed GMPL submissions can be retried from the Exported Batches screen; Excel is rebuilt from batch data (no manual file upload required). |
| **Manual fallback** | Admin can upload a supplier-format Excel file directly to GMPL if needed. |
| **External API** | Partner file orders (MTN only) can submit to GMPL via the same endpoint. |

**Business rules on Kellishub side:**

- GMPL submission is **MTN only** (`network_provider=mtn`).
- Telecel / Airtel Tigo orders are exported internally but **not** sent to GMPL.

### 3.3 Excel file format (generated by Kellishub)

Kellishub builds `.xlsx` files with two columns:

| Column A | Column B |
|----------|----------|
| `Phone Number` | `Data Size` |

- Phone numbers are normalized (e.g. `233XXXXXXXXX` → `0XXXXXXXXX` where applicable).
- Data size is taken from the product bundle description (numeric portion).
- Sheet name: `Orders`.

**Please confirm** this matches GMPL's required template (column names, sheet name, and any extra columns).

### 3.4 Status sync (assumed — not in original curl doc)

Kellishub also attempts to poll order status (for auto-marking orders **Completed**) using:

- `GET {GMPL_API_URL}/api/orders/agents/{id}`
- `GET {GMPL_API_URL}/api/orders/agents/{id}/status`

Optional webhook endpoint on Kellishub:

- `POST https://api.kellishub.com/order/gmpl/webhook`
- Optional shared secret header: `x-gmpl-webhook-secret` or `GMPL_WEBHOOK_SECRET`

**Please confirm** whether these endpoints exist, their exact paths, response format, and whether GMPL supports webhooks.

## 5. Test results — connectivity failure

### 5.1 Production server (Render)

Kellishub API logs repeatedly show:

```text
[exportPendingOrders] GMPL submit failed: getaddrinfo ENOTFOUND api.gmpl.com
[submitAdminGmplFile] getaddrinfo ENOTFOUND api.gmpl.com
```

`ENOTFOUND` means DNS lookup failed — the hostname cannot be resolved.

### 5.2 Local testing (Windows)

```text
C:\Users\threadace> nslookup api.gmpl.com
*** UnKnown can't find api.gmpl.com: Non-existent domain

C:\Users\threadace> curl -I https://api.gmpl.com/api/orders/agents/new
curl: (6) Could not resolve host: api.gmpl.com
```

**Conclusion:** `api.gmpl.com` is **not registered or not published in public DNS** at the time of testing. This is not an authentication or payload formatting error; no TCP connection to GMPL is possible.

### 5.3 Impact on Kellishub

- Batches are created and orders move to **Processing** locally.
- Excel files can be downloaded from the admin dashboard.
- **GMPL submit always fails** → batches show **GMPL Failed** in the UI.
- Auto-export, manual retry, and manual upload all fail with the same DNS error.
